# Product

## Register

product

## Platform

web

## Users

Solo operators and power users running many parallel AI coding agents on their own machine. They juggle Git worktrees, durable tmux sessions, and agent CLIs, and open Cockpit when they need to stay oriented without losing track of what needs attention.

## Product Purpose

Ajax is the operator layer for that fleet: one place to see what is active, what needs input, what is ready to review, and which action is safe next. Success means staying in control—never missing an approval or a stuck task—while still shipping faster than raw git and tmux alone. Control and attention are the primary win; speed follows from that clarity.

## Positioning

A mobile-first remote cockpit for your local agent fleet: Safari on the phone, the host does the work.

## Brand Personality

Fast, decisive, under-control. The interface should feel like an operator console—immediate status, clear next action, no ceremony. Voice stays direct and operational; emotion is composure under load, not playful delight.

## Anti-references

- Generic SaaS dashboards: card grids, metric strips, soft purple/indigo chrome.
- Overbuilt IDE shells: too many panels and tabs fighting the terminal.

## Design Principles

1. **Terminal is the work surface** — chrome supports the task; it never competes with the pane.
2. **Status and next safe action beat decoration** — every screen should answer what is happening and what to do.
3. **Mobile-reachable without diluting host authority** — phone access is first-class; task truth stays on the host.
4. **Decisive actions over exploratory chrome** — prefer clear operator intents over browsing UI for its own sake.

## Accessibility & Inclusion

No formal accessibility requirements for now.
