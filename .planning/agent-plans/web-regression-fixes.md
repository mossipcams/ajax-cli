# TDD Implementation Packet — web regression fixes

## Goal

Make the fullscreen ⛶ expand in `TerminalRawView.svelte` resize the terminal grid
to the full-bleed panel width **even while the iOS keyboard is open**, via a
one-shot `expandFlushPending` exemption mirroring the existing `pinchFlushPending`.
Then lock the two already-applied sibling fixes (NewTaskSheet agent segmented
pills; Drop action → dashboard) with tests.

Root cause of the reported bug: entering expand calls `focusTerm()`, which pops
the iOS keyboard; `fitNow()`'s keyboard-open guard then early-returns and skips
the grid resize, so the canvas stays at its narrower pre-expand column count,
left-aligned, leaving an empty column down the right edge (where the ⛶ button is).

## Architecture context

`TerminalRawView.svelte` owns the follow-output / PTY-lockstep policy. The
keyboard-open guard exists because a tmux-attached client SIGWINCHes the shared
window on every resize; spraying resizes during the iOS keyboard animation
corrupts the pane. `pinchFlushPending` is the established precedent for a
deliberate one-shot grid+PTY resize while the keyboard is open (pinch-end refits
grid and PTY in the same pass, so lockstep holds). Expand is the same shape of
discrete, intentional layout change, so it takes the same exemption.

`isKeyboardOpen()` (from `../viewport`) reads the `keyboard-open` class on
`document.documentElement` — tests simulate keyboard-open by adding that class.
