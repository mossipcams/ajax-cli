## Context evidence

**Desired behavior — Escape dismiss.**
`NewTaskSheet.tsx:136-138`, `onKeyDown` on the dialog element:

```tsx
onKeyDown={(event) => {
  if (event.key === "Escape") onClose?.();
}}
```

The handler is bound to the element carrying `data-testid="new-task-sheet"`
(`NewTaskSheet.tsx:128-139`). React's synthetic `keydown` bubbles, so firing on
that element or any descendant reaches it.

**Desired behavior — backdrop dismiss with an inside-click guard.**
`NewTaskSheet.tsx:121-123`:

```tsx
function handleBackdropClick(event: MouseEvent<HTMLDivElement>) {
  if (event.target === event.currentTarget) onClose?.();
}
```

Bound as `onClick` on the same dialog element (`NewTaskSheet.tsx:135`). The
`target === currentTarget` guard is the whole behavior: a click landing on the
dialog element itself dismisses; a click landing on the `<form class="sheet-card">`
(`NewTaskSheet.tsx:140-145`) or anything inside it bubbles up with a different
`target` and must NOT dismiss.

**Existing test patterns to reuse.** `NewTaskSheet.test.tsx`:
- imports, line 1-8: `describe, it, expect, vi, afterEach` from `vitest`;
  `render, fireEvent` from `@testing-library/react`. All needed symbols are
  already imported — add none.
- fixture, line 13: `const repos = [{ name: "web" }, { name: "api" }];`
- cleanup, line 15: `afterEach(() => vi.restoreAllMocks());`
- the `onClose` + `vi.fn()` + `container.querySelector` idiom, lines 77-93
  ("dismisses when the grabber is dragged down past the threshold") — closest
  analog; reuse its shape.
- `getByTestId` idiom, lines 18-21.

**Boundaries.** Pure component-level vitest/jsdom. No API, no network, no
localStorage, no module boundary crossed. `vi.spyOn(api, ...)` is unnecessary
because neither dismiss path calls `startTask`.
