## Global Constraints (apply to every packet)

- **Mobile-first operator.** The user drives this over SSH from an iOS terminal. No
  keyboard-chord shortcuts; touch targets ≥ 44px.
- **Served assets are the BUILT bundle.** `crates/ajax-web/src/adapters/assets.rs` embeds
  `crates/ajax-web/web/dist/{index.html,app.js,app.css}` via `include_str!`/`include_bytes!`.
  Any `.svelte`/`.ts`/`app.html` edit is invisible to Rust until you run `npm run web:build`,
  which regenerates `web/dist/*`. **Always rebuild before running Rust tests.**
- **Web-asset assertions.** `crates/ajax-cli/src/web_backend.rs` and
  `crates/ajax-web/src/slices/install.rs` assert on substrings of the built bundle. Editing
  `web/*` can trip them — only Packet E intends to.
- **Worktree bootstrap.** This worktree has no `node_modules`. Run `npm install` once at the
  repo root before any `web:*` script.
- **Rust toolchain pinned to 1.88.0.** Tests run single-threaded:
  `cargo nextest run --all-features --test-threads=1`.
- **Commits:** Conventional Commits. This feature is `feat:`. **Never** add `Co-Authored-By`
  or "Generated with" trailers (user global rule). Do not put "Claude" in commit subjects.
- All web commands below run **from the repo root** (`crates/ajax-cli__worktrees/ajax-web-ux-improvements`).

### Architecture context (shared)

```
crates/ajax-web/
  web/src/components/   Svelte UI (TerminalPanel = live xterm bridge; PanePanel = legacy poll)
  web/src/{api,types,contracts,state,polling}.ts   transport + pure projection helpers
  src/slices/           behavior (cockpit, operate, terminal, pane, install) — no IO
  src/adapters/         IO (terminal_pty = websocket↔PTY, assets, server, tls, http)
```

Dependency direction: `runtime` → `slices` + `adapters`; `slices` never call axum.
The **live terminal** path is `runtime::axum_task_terminal` → `adapters::terminal_pty::
bridge_task_terminal_socket` (a real PTY running `tmux attach-session`). The **legacy poll**
path is `runtime::axum_task_pane/answer/input` → `slices::pane` (tmux `capture-pane` +
`send-keys`). Packets A–D improve the live path; Packet E deletes the legacy path.

> ⚠️ `tmux capture-pane` is **also** used by the cockpit refresh / agent-activity detection
> (`crates/ajax-cli/src/{cockpit_backend,agent_status_cache}.rs`). Packet E must **not**
> touch those. Only the web `/pane`, `/answer`, `/input` bridge is removed.

## Packet C — On-screen key bar for terminal control keys

### 1. Goal

## Packet D — Harden mobile text input

### 1. Goal
Stop the iOS keyboard from corrupting terminal input: disable autocapitalize/autocorrect/
autocomplete/spellcheck on xterm's hidden textarea, and focus the terminal once connected so
the keyboard is one tap away.

### 2. Allowed files
- Production: `crates/ajax-web/web/src/components/TerminalPanel.svelte` (`<script>` only)
- Test: `crates/ajax-web/web/src/components/TerminalPanel.test.ts`

### 3. Forbidden changes
- Do not set `inputmode`/`enterkeyhint` hacks or replace xterm's textarea.
- Do not change echo/zerolag logic.

### 4. Architecture context
xterm renders a visually-hidden `<textarea>` it owns at `term.textarea`. Mobile browsers apply
autocorrect/autocapitalize to it, which mangles commands. Setting the attributes after
`term.open(container)` fixes it. `term.focus()` on socket `open` brings up the keyboard.

### 5. Code anchors
- `TerminalPanel.svelte:35-38` — `if (container) { term.open(container); ... }`
- `TerminalPanel.svelte:64-67` — socket `open` handler
- `MockTerminal` (`TerminalPanel.test.ts:18-29`) has no `textarea`/`focus` — extend in the test.

### 6. Test-first instructions

```ts

```

